✍️ 번역문

이슈란 무엇인가?

“이슈”란 말 그대로 이슈를 설명하기 위한 깃허브 게시물을 의미하는 일반적인 용어입니다. 이슈는 버그 리포트, 새로운 기능 추가 요청, 토론 등 p5.js 라이브러리 개발과 관련되어 있는 것이라면 어떤 형태의 게시물도 될 수 있습니다. 깃허브 계정을 가지고 있다면 누구나 각 이슈에 댓글을 달 수 있습니다. 물론 봇들도요! 정리하자면 이슈는 리포지터리 내에서 개발과 관련된 주제를 토론할 수 있도록 만들어진 공간입니다.

이슈는 다양한 이유로 열릴 수 있지만, 우리는 주로 p5.js 소스 코드 개발과 관련해서만 이슈를 열고 있습니다. 여러분의 코드를 디버깅하거나, 여러분의 프로젝트에 공동 작업자를 초대하는 것과 같이 소스 코드 개발과 관련 없는 주제는 포럼이나 디스코드 같은 다른 플랫폼에서 논의해주세요.

여러분이 논의하고자 하는 주제가 깃허브 이슈에 해당하는지 아니면 다른 곳에 게시되어야 하는지 판단하는데 도움을 드리기 위하여 사용하기 쉬운 이슈 템플릿을 만들었습니다.

⚠️ 비고/논의사항

- 번역문(원문) 형태로 써야 하는 용어의 기준을 잡아주시면 다음 번역에는 반영해보겠습니다.
- 링크 역시 어떤 기준으로 넣어야 하는지 논의가 필요할 것 같아요.(포럼, 디스코드 등)

### 염인화

✍️ 번역문

이슈란 무엇인가?

"이슈(issue)"란 깃허브(GitHub) 게시물의 하나로, 어떤 문제나 논의사항을 지칭하는 일반적인 표현입니다. 이슈는 버그 리포트, 새 기능 추가 요청, 토론, 또는 p5.js 라이브러리 개발과 관련된 모든 게시물 등을 포함합니다. 누구나 깃허브 계정만 있다면 이슈 게시물에 댓글을 남길 수 있습니다. 심지어 봇들도요! 다시말해, 이슈는 기여자들이 해당 리포지토리(repo) 내에서 프로젝트 개발 관련 주제들을 논의할 수 있는 곳입니다.

이슈는 다양한 목적을 위해 게시(open)될 수 있으나, 대체로 p5.js 소스 코드 개발 논의를 위해서만 활용됩니다. 개별 코드 디버깅, 프로젝트 공동 작업자 모집, 그 외 무관한 주제들은 [포럼(forum)](https://discourse.processing.com/)과 [디스코드(Discord)](https://discord.gg/SHQ8dH25r9)와같은 다른 플랫폼에서 논의가능합니다.

여러분이 게시하고자 하는 내용이 깃허브 이슈에 해당하는지, 아니면 다른 플랫폼에 게시되어야 하는지 여부를 판단하는 데 도움이 되고자, 이 쓰기 쉬운 "이슈 템플릿"을 만들었습니다.


⚠️ 비고/논의사항

- 업계에서 자주쓰이는 영어 고유명사나 브랜드명은 음차번역하였고, 그 경우 영어 원문 표기를 괄호 안에 병기하였습니다.
- 깃허브에 아직 익숙치 않은 초심자를 고려하여, 필요에따라 약간의 부연 표현/구절을 추가하기도 했습니다.
- 한국어/영어 문장 구조가 다른 걸 고려하여, 가독성 향상을 위해 문장을 임의로 나누기도 했습니다.
- "리포지토리"는 국립국어원 홈페이지에서는 검색이안돼서 구글 검색을 통해 확인했습니다.

이슈는 다양한 이유로 등록(Open)될 수 있지만, 보통은 p5.js 소스 코드의 개발과 관련된 논의를 위해서만 이슈를 사용합니다. 개인의 코드 디버깅, 프로젝트 협력자 모집 또는 기타 관련이 없는 주제들은 [포럼](https://discourse.processing.com/)이나 [Discord](https://discord.gg/SHQ8dH25r9)와 같은 다른 플랫폼에서 논의되어야 합니다.

여러분이 게시하고자 하는 주제가 깃허브 이슈에 해당하는지, 아니면 다른 곳에 게시되어야 하는지 결정하는 데 도움이 될 수 있도록 쓰기 쉬운 “이슈 템플릿”을 만들어 놓았습니다!


⚠️ 비고/논의사항

- “이슈(issue)를 오픈(Open)한다”는 문장이 “글을 등록하다”로 해석되는게 맞아보이나.. 깃허브 세계관상으로는 “이슈를 열고 닫는다”는 의미를 내포하는게 좋을 것 같아서 이 부분이 고민이 됩니다.


### 이현우

✍️ 번역문

이슈(Issues)란 무엇인가요?

“이슈(Issue)”는 논쟁에 대해 설명하는 것을 목표로 하는 GitHub 게시글의 총칭입니다. 이 이슈는 버그 리포트, 새 기능 추가에 대한 요청, 논의와 같은 p5.js 라이브러리 개발과 관련된 어떤 내용도 담을 수 있습니다. GitHub 계정을 가지고 있다면 누구나 각 이슈에 대해 댓글을 남길 수 있습니다. 심지어 봇도 가능해요! 이곳은 리포지토리(repo)에서 기여자들이 프로젝트 개발과 관련된 주제를 토론하는 곳입니다.

비록 매우 다양한 이유로 이슈를 게시(open)할 수 있겠지만, 보통은 p5.js 소스 코드 개발에 대해 논의하는 용도로만 사용합니다. 자신의 코드를 디버깅하거나, 프로젝트에 공동 작업자를 모집하거나 기타 관련 없는 주제는 [포럼](https://discourse.processing.com/)이나 [Discord](https://discord.gg/SHQ8dH25r9)와 같은 다른 플랫폼에서 논의하세요.

주제가 GitHub 이슈로 게시되어야 하는지 아니면 다른 곳에 게시되어야 하는지 결정하는 데 도움이 되는 사용하기 쉬운 이슈 템플릿을 만들었습니다!

⚠️ 비고/논의사항
- repo : 원문에서 이미 repository를 repo로 줄여서, 이를 한국어에서도 “리포”와 같이 줄여야 하는지.. 잘 쓰지는 않는 듯합니다. 그리고 레포지토리 레파지토리 리파지토리 등 단어를 통일해야 할 것 같습니다. GitHub Docs에 의하면 공식 단어는 리포지토리입니다.
- invite : 사전적 정의는 ‘초대하다’가 맞지만 맥락상 모집이 조금 더 적합하다고 생각하는데.. 여러 의견이 있을 것 같습니다.
- issue : 처음 이슈를 정의할 때만 사전적 정의로 하고 이외에는 GitHub Issues를 지칭하는 것이라고 판단하고 그냥 ‘이슈’로 번역했습니다.
- 각 플랫폼(GitHub, Discord)은 브랜드명이 영어이기 때문에 원문 그대로 넣었습니다.
- repo, open은 GitHub에서 사용하는 고유 용어로 판단, 원문을 병기하였습니다.

 ⚠️ 비고/논의사항
 - repo : 리포, 리포지토리, 저장소 단어는 ‘리포지토리’로 적어두었습니다.
